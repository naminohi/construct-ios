✨ KEY INSIGHTS YOU'LL LEARN:
═══════════════════════════════════════════════════════════════════════════════════

1. Current design is STATELESS from Swift's perspective
   → Each function independent, parameters explicit
   → Good isolation, no cross-app interference

2. Thread-safety is BUILT-IN via Arc<Mutex<>>
   → Safe for concurrent access
   → No poisoned lock issues
   → All types implement Send + Sync

3. Session storage is currently MANUAL
   → Must call export_session_json() to Keychain
   → Must call import_session_json() from Keychain
   → Gap: No automatic sync on changes

4. Callbacks are ONE-WAY (Rust → Swift)
   → Rust can call Swift methods
   → Swift cannot call back into Rust
   → Synchronous (blocks during operation)

5. Per-instance design ENABLES multi-device support
   → Each device gets its own Arc<ClassicCryptoCore>
   → Coordinator can be added at app layer
   → No changes needed to Rust core
